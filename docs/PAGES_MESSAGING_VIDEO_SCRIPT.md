## ğŸ¥ Recording Steps

### SCENE 1: Login (0:00 - 0:20)

**Actions:**
1. Go to `https://asm-frontend-omega.vercel.app`
2. Show the login page
3. Enter test account email
4. Enter password
5. Click "Sign In"
6. Brief view of dashboard

**Say (optional narration):**
> "I'm logging into our social media management platform with the test account. This is the main dashboard where users can manage their social media presence."

---

### SCENE 2: Navigate to Inbox (0:20 - 0:40)

**Actions:**
1. Show the sidebar navigation
2. Hover over "Inbox" menu item
3. Click "Inbox"
4. Wait for page to load

**Say:**
> "Now I'm navigating to the Inbox feature where users can manage their Facebook Messenger conversations. Let's see what's loaded."

---

### SCENE 3: Display Conversations (0:40 - 1:30)

**Actions:**
1. Point out (with cursor) the conversation list on the left
2. Hover over each conversation to show details:
   - Customer name
   - Facebook Messenger icon
   - Message preview
   - Timestamp
   - Unread count (if any)
3. Show there are multiple conversations
4. Show the conversations are sorted by most recent

**Say:**
> "Here's our unified inbox showing all Facebook Messenger conversations. You can see customer names, message previews, timestamps, and platform indicators. These are real messages from our test Page. Notice the Facebook icon showing these are Messenger conversations."

**Visual Focus:**
- Slowly move cursor down the conversation list
- Pause on each conversation for 2-3 seconds
- Make sure text is readable in the recording

---

### SCENE 4: Open Conversation (1:30 - 2:30)

**Actions:**
1. Click on the first conversation
2. Show the message thread loading
3. Point out the conversation header:
   - Customer profile
   - Platform name (Facebook)
4. Scroll through the message history:
   - Show customer messages (left side)
   - Show any existing replies (right side)
5. Show timestamps on messages
6. Show the reply input box at the bottom

**Say:**
> "I'm opening a conversation to view the full message thread. You can see the customer's messages on the left and any business replies on the right. Each message has a timestamp. This is actual data from Facebook Messenger using the pages_messaging permission."

**Visual Focus:**
- Let the message thread fully load
- Scroll slowly if there are many messages
- Pause to show message details clearly

---

### SCENE 5: Send Reply (2:30 - 3:30)

**Actions:**
1. Click in the reply input box
2. Type a professional response slowly (so reviewer can read):
   ```
   Thank you for reaching out! We're happy to help. How can we assist you today?
   ```
3. Show the "Send" button
4. Click "Send"
5. Show the message appearing in the thread
6. Show the message on the right side (business reply)
7. Show the timestamp updating

**Say:**
> "Now I'm sending a reply to this customer. I'll type a response and click Send. Watch as the message appears in our conversation thread. This sends directly to Facebook Messenger using the pages_messaging permission."

**Visual Focus:**
- Type slowly enough to be readable
- Pause after sending to show the message in the thread
- Make sure the new message is fully visible

---

### SCENE 6: Verify in Facebook Messenger (3:30 - 4:30)

**Actions:**
1. Open a new browser tab
2. Go to `facebook.com`
3. Navigate to your Page (or direct to Messenger)
4. Click on Messages/Inbox
5. Find the same conversation
6. Show that your reply from the app appears in Facebook Messenger
7. Point out the message text matches
8. Show the timestamp

**Say:**
> "Let me verify this message actually sent to Facebook Messenger. I'm opening Facebook in a new tab and going to the Page's Messenger inbox. As you can see, the reply we just sent from our platform appears here in Facebook Messenger. This confirms the pages_messaging permission is working correctly."

**Visual Focus:**
- Make it very clear you're on facebook.com
- Show the URL bar
- Highlight the matching message text
- Show it's the same conversation

---

### SCENE 7: Additional Features (4:30 - 5:00)

**Actions:**
1. Go back to your app tab
2. Click on a different conversation
3. Show the messages load
4. Show you can switch between conversations
5. (Optional) Show the unread count feature
6. Brief view of the full inbox interface

**Say:**
> "Users can seamlessly switch between multiple customer conversations. Our platform consolidates all Facebook Messenger communications in one place, making customer service more efficient. This is the core functionality enabled by the pages_messaging permission."

**Visual Focus:**
- Click confidently between conversations
- Show smooth navigation
- End with a full view of the inbox interface

---

### ENDING (5:00)

**Actions:**
1. Briefly show the inbox one more time
2. Hover cursor over key features
3. Stop recording

**Say:**
> "This completes the demonstration of how our app uses the pages_messaging permission to manage Facebook Messenger conversations. Thank you for reviewing our submission."

- Check the message isn't empty
- Ensure Facebook token is valid
- Try refreshing and resending

- Wait 30 seconds for sync

- Check you're viewing the correct Page inbox
